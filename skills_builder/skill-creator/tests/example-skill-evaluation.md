# Example Skill Creator Evaluation Notes

These are starter notes for testing `skill-creator`. They are intentionally small. Real projects should expand the eval set once the skill direction settles.

## Eval 1: New skill from rough idea

Checks whether Skill Creator can:

- capture intent
- identify trigger conditions
- suggest output format
- propose realistic starter evals
- avoid drowning a non-technical user in jargon

## Eval 2: Existing skill improvement

Checks whether Skill Creator can:

- preserve the original skill name
- avoid unnecessary rewrite-from-zero behaviour
- identify trigger weaknesses
- propose description optimisation
- suggest evals for should-trigger and should-not-trigger cases

## Eval 3: Casual/light mode request

Checks whether Skill Creator can:

- respect the user's request to skip full benchmarking
- still improve the skill sensibly
- avoid pretending a light pass equals rigorous validation
